盒子模型

margin塌陷现象
如果一个大盒子中包含一个小盒子，在给小盒子设置margin-top时，大盒子也会一起向下平移。
解决方案：
(1)给大盒子加一个边框
(2)给大盒子设置一个overflow属性
(3)添加浮动

margin的外边距合并现象
当两个盒子都设置margin-top和margin-bottom时，实际距离会取值
为较大的一方。在实际运用中，常常将margin-bottom是为较大值。

浮动的特点：
(1)只影响他后面的元素
(2)浮动不占标准流位置
(3)浮动找浮动，不浮动找不浮动(既是两者紧挨着)
(4)浮动显示的位置与原本不浮动的位置是对应的
(5)浮动会改变元素的显示方式(设置浮动后，可设置元素的宽高)
清浮动的解决方法：
(1)给有浮动的元素的父元素添加overflow：hidden
(2)给有浮动的元素的父元素添加一个伪类，具体代码如下：

  • .clearfix:after{
  • content:””;//添加内容为空
  • height:0;//内容高度为0
  • line-height:0;//内容文本高度为0
  • display:block;//将文本设置为块级元素
  • clear:both;//清除浮动
  • visibility:hidden;//将元素隐藏
  • }
  • .clear{
  • zoom:1;//兼容ie6
  • }

(1)相对定位(relative)：如果设置相对定位，盒子将会在原来的位置上进行平移，设置相对定位的元素在页面上占据了位置即使没有脱离标准流。
(2)绝对定位(absolute):具有绝对定位的元素根据具有定位的父元素来定位(包含上上级元素)，如果都没有，这相对body来定位。绝对定位的元素在页面上不会占据了位置即是脱离标准流。

(3)固定定位(fixed):不管页面多大，固定定位永远是相对于浏览器的边框来说的。固定定位的元素在页面上不会占据了位置即是脱离标准流。
